git init -> torna a pasta versionavel oq torna ela versionavel é uma pasta que fica dentro dela (.git (invisível) pra ver basta dar um ls -a)

git status -> mostra literalmente os status (qual branch, quantos commits, arquivos que ainda não foram trackeados (não foram atualizados para participar do versionamento)

git add "nome do arquivo" ou pode usar " . " (esse ponto adiciona todos os arquivos do diretorio atual {quando você usar "ls -a" voce vera que tera esse ./ e a ../ que é a pasta mae de todo o diretório}) -> adiciona o que nos queremos que vai para o commit, prepara e elenca oq vai entrar no commit

git commit -m "mensagem" -> Criando um checkpoint dando uma descrição do save

IDENTIFICAO PARA "AASSINAR" OS COMMITS
(git config --global user.email "seu email", git commit --user.name "seu user name")

git log -> mostra todo o histórico de commits

#FLUXO DE TRABALHO:
1. criar o diretório (pasta) do projeto
2. iniciar o git dentro do diretório criado (git init .)
3. edita ou cria arquivos
4. git status
5. git add <nome aquivo>
6. git status
7. git commit -m "mensagem"
8. git status
9. volta para o passo 3

------------------------------------------------------------------------------------------------------

git diff <nome arquivo> -> mostra dentro do arquivo oq ha de novo no arquivo

git reset <nome do arquivo> -> tira o estado atual do arquivo (tira ele do commit)

*BRANCHES* (Ramificacoes)

git checkout -b <nome da npva branch> -> cria uma nova branch e ja muda pra ela 

{
leolo@notebook MINGW64 ~/Downloads/curso_git_2026 (main) <--
$ git checkout -b secundaria				 <--
Switched to a new branch 'secundaria'			 <-- switched to secundaria (nova branch)

leolo@notebook MINGW64 ~/Downloads/curso_git_2026 (secundaria) <---- veja que já mudou a branch
$ git status
On branch secundaria 		<----- perceba
nothing to commit, working tree clean
}

git branch -> mostra todas as branches existentes e qual sua atual branch

git checkout <nome da brainch> -> serve para mudar de branch

quando se cria uma branch nova você basicamente clona tudo que tem na main(anterior).

para trazer o que esta na branch secundaria (nova branch criada) :

git merge <nome da branch que você quer misturar> -> isso faz com que as ALTERACOES que estão numa outra branch entre na branch atual na qual você esta durante o merge

----------------------------------------------------------------------------------------
EXEMPLO:
branch atual = main
branch que vai se misturar = secundaria 

leolo@notebook MINGW64 ~/Downloads/curso_git_2026 (main) <--- branch atual
$ git merge secundaria <--- nome da branch que quero fundir na main

----------------------------------------------------------------------------------------

git branch -d <nome da branch a ser deletada> --> deleta uma branch

*CONFLITOS*:

O conflito visto no exemplo do teo foi: duas branches que modificaram A MESMA LINHA DO MESMO ARQUIVO (não tem problema modificar o mesmo arquivo e sim a mesma linha), para resolver esse conflito vc deve LITERALMENTE entrar no arquivo que esta dando conflito , e manualmente modificar a linha do jeito que você achar melhor e depois fazer o (add . ; commit -m,).
